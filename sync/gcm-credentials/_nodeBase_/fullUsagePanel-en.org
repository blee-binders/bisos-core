** +
GCM (Git Credential Manager) stores and retrieves HTTPS credentials for Git.
On headless Debian 12 (no GUI keyring), GPG is the recommended credential store.
GCM encrypts tokens/passwords with your GPG key — no plaintext on disk.

** Scope: GnuPG + pass BackStore prerequisites for Git Credential Manager.

** Layering and order:
  1) Run this layer first:
       gpg-sbom.pcs -i sbom_fullUpdate
  2) Do one-time key/store init as target user:
       gpg --full-generate-key
       gpg --list-secret-keys --keyid-format LONG
       pass init <gpg-id>
  3) Then run git layer:
       git-sbom.pcs -i sbom_fullUpdate

** How this layer fits:
  - Provides gnupg, pass, and pinentry programs.
  - Prepares the encrypted BackStore used later by git-sbom/GCM.
  - Has no dependency on git package setup.

** Pinentry guidance:
  - This layer uses Debian-safe pinentry packages: pinentry-curses and pinentry-tty.
  - Emacs/Magit users should use loopback fallback when terminal pinentry conflicts.
  - For terminal/headless sessions, pinentry-curses is preferred.
