####+BEGIN: blee:bxPanel:foldingSection :outLevel 1 :title "Overview and Use Cases" :anchor "Panel" :extraInfo "  /GeForce RTX 5060 Ti -- Debian 13/"
* [[elisp:(show-all)][(>]]  _[[elisp:(blee:menu-sel:outline:popupMenu)][±]]_  _[[elisp:(blee:menu-sel:navigation:popupMenu)][Ξ]]_       [[elisp:(outline-show-subtree+toggle)][| *Overview and Use Cases:* |]] <<Panel>>   /GeForce RTX 5060 Ti -- Debian 13/  [[elisp:(org-shifttab)][<)]] E|

This panel describes how to install the *NVIDIA proprietary driver* for a
*GeForce RTX 5060 Ti* (Blackwell, =GB206=) on *Debian 13 (Trixie)*.

*The key point up front:* the RTX 5060 Ti is a *Blackwell*-generation card. Blackwell GPUs
require an NVIDIA driver of the *570 series or newer* AND the *open kernel modules* (MIT/GPL).
The driver that ships in the base Debian 13 repository (=550.xx=) does *not* support Blackwell
and will leave you on =nouveau= or a black screen. So the ordinary =apt install nvidia-driver=
recipe (as used in the older =hwGraphicsNvidia= panel) *does not work on this card*.

*Approach used by this panel:*
- Primary  :: Install the *open-module* driver (=nvidia-open-kernel-dkms=) from *trixie-backports*,
  which carries the newer (570+) driver that supports Blackwell.
- Secure Boot :: Left *enabled* --- we enroll a Machine Owner Key (MOK) so the DKMS-built module is signed and loads.
- Fallback  :: If backports does not yet carry a 570+ driver, use NVIDIA's official =.run= installer
  with the open kernel modules (last section).
- GPU scope :: Treats the NVIDIA card as the primary GPU. (This machine also has Intel Arrow Lake-S
  integrated graphics; hybrid PRIME render-offload is intentionally *not* covered here.)

/NOTE: This is a starting-point draft. Blackwell support on Debian is evolving fast; verify each
step and the current backports driver version against this specific machine before relying on it./

####+BEGIN: blee:bxPanel:foldingSection :outLevel 1 :title "This Card -- Why the Packaged Driver Fails" :anchor "" :extraInfo "Blackwell -- GB206"
* [[elisp:(show-all)][(>]]  _[[elisp:(blee:menu-sel:outline:popupMenu)][±]]_  _[[elisp:(blee:menu-sel:navigation:popupMenu)][Ξ]]_       [[elisp:(outline-show-subtree+toggle)][| *This Card -- Why the Packaged Driver Fails:* |]]  Blackwell -- GB206  [[elisp:(org-shifttab)][<)]] E|
####+END

The card, as reported by =lspci -nn= on the target machine:

#+begin_example
01:00.0 VGA compatible controller [0300]: NVIDIA Corporation GB206 [GeForce RTX 5060 Ti] [10de:2d04] (rev a1)
01:00.1 Audio device [0403]: NVIDIA Corporation GB206 High Definition Audio Controller [10de:22eb] (rev a1)
#+end_example

- *Architecture* :: Blackwell (=GB206=); PCI vendor:device =10de:2d04=.
- *Driver floor* :: Blackwell needs the *570-series* driver (or newer). Anything older simply has no
  device-ID match for =GB206= and will not drive the card.
- *Open modules are mandatory* :: For Turing and newer, NVIDIA offers two kernel-module flavors ---
  the legacy *proprietary* module and the *open* (MIT/GPL) module. For *Blackwell the open module is
  required*; the proprietary/closed kernel module refuses to initialize on these GPUs.
- *Why base Trixie fails* :: Debian 13's stock =nvidia-driver= is =550.xx= --- pre-Blackwell and
  closed-module. Hence we pull the open-module driver from *trixie-backports* (or use the =.run= fallback).

####+BEGIN: blee:bxPanel:foldingSection :outLevel 1 :title "Prerequisites and Assumptions" :anchor "" :extraInfo "Debian 13 -- UEFI -- Secure Boot ON"
* [[elisp:(show-all)][(>]]  _[[elisp:(blee:menu-sel:outline:popupMenu)][±]]_  _[[elisp:(blee:menu-sel:navigation:popupMenu)][Ξ]]_       [[elisp:(outline-show-subtree+toggle)][| *Prerequisites and Assumptions:* |]]  Debian 13 -- UEFI -- Secure Boot ON  [[elisp:(org-shifttab)][<)]] E|
####+END

- A working *Debian 13 (Trixie)* install, booted in *UEFI* mode, with network access.
- *Secure Boot is ON* --- we will enroll a MOK so the signed NVIDIA module loads (Step 5).
- =sudo=/root access.
- Enough free space for the driver and its DKMS builds (a few hundred MB).
- A plan for console access in case X/Wayland fails to start after install (another machine + SSH,
  or comfort with a text console via =Ctrl-Alt-F3=).

####+BEGIN: blee:bxPanel:foldingSection :outLevel 1 :title "Step 1 -- Confirm the Card and Current State" :anchor "" :extraInfo "lspci"
* [[elisp:(show-all)][(>]]  _[[elisp:(blee:menu-sel:outline:popupMenu)][±]]_  _[[elisp:(blee:menu-sel:navigation:popupMenu)][Ξ]]_       [[elisp:(outline-show-subtree+toggle)][| *Step 1 -- Confirm the Card and Current State:* |]]  lspci  [[elisp:(org-shifttab)][<)]] E|
####+END

Confirm the GPU is present and see which driver is currently bound:

#+begin_example
# Identify the GPU (expect: NVIDIA Corporation GB206 [GeForce RTX 5060 Ti] [10de:2d04])
lspci -nn | egrep -i 'VGA|3D|NVIDIA'

# See what kernel driver is in use (likely "nouveau" before we start)
lspci -k -d 10de:2d04
#+end_example

####+BEGIN: blee:bxPanel:foldingSection :outLevel 1 :title "Step 2 -- Enable non-free-firmware and trixie-backports" :anchor "" :extraInfo "APT sources"
* [[elisp:(show-all)][(>]]  _[[elisp:(blee:menu-sel:outline:popupMenu)][±]]_  _[[elisp:(blee:menu-sel:navigation:popupMenu)][Ξ]]_       [[elisp:(outline-show-subtree+toggle)][| *Step 2 -- Enable non-free-firmware and trixie-backports:* |]]  APT sources  [[elisp:(org-shifttab)][<)]] E|
####+END

The NVIDIA driver lives in =non-free=; firmware in =non-free-firmware=. The Blackwell-capable
driver comes from =trixie-backports=.

Ensure =/etc/apt/sources.list= has all components:

#+begin_example
deb http://deb.debian.org/debian/ trixie main contrib non-free non-free-firmware
deb http://security.debian.org/debian-security/ trixie-security main contrib non-free non-free-firmware
#+end_example

Add backports in =/etc/apt/sources.list.d/backports.list=:

#+begin_example
deb http://deb.debian.org/debian/ trixie-backports main contrib non-free non-free-firmware
#+end_example

Pin the kernel/firmware/nvidia packages to backports so the driver and kernel stay in step ---
create =/etc/apt/preferences.d/nvidia-backports.pref=:

#+begin_example
Package: linux-* libcpupower* firmware-* nvidia-*
Pin: release n=trixie-backports
Pin-Priority: 500
#+end_example

Then refresh and take the newer kernel + headers from backports:

#+begin_example
sudo apt update
sudo apt full-upgrade
#+end_example

####+BEGIN: blee:bxPanel:foldingSection :outLevel 1 :title "Step 3 -- Install Kernel Headers" :anchor "" :extraInfo "DKMS build deps"
* [[elisp:(show-all)][(>]]  _[[elisp:(blee:menu-sel:outline:popupMenu)][±]]_  _[[elisp:(blee:menu-sel:navigation:popupMenu)][Ξ]]_       [[elisp:(outline-show-subtree+toggle)][| *Step 3 -- Install Kernel Headers:* |]]  DKMS build deps  [[elisp:(org-shifttab)][<)]] E|
####+END

DKMS compiles the NVIDIA module against your running kernel, so it needs matching headers:

#+begin_example
sudo apt install -t trixie-backports linux-headers-amd64
#+end_example

/If you did not pin kernels to backports in Step 2, install the headers that match =uname -r=
instead: =sudo apt install linux-headers-$(uname -r)=./

####+BEGIN: blee:bxPanel:foldingSection :outLevel 1 :title "Step 4 -- Install the Open-Module Driver from Backports" :anchor "" :extraInfo "nvidia-open-kernel-dkms"
* [[elisp:(show-all)][(>]]  _[[elisp:(blee:menu-sel:outline:popupMenu)][±]]_  _[[elisp:(blee:menu-sel:navigation:popupMenu)][Ξ]]_       [[elisp:(outline-show-subtree+toggle)][| *Step 4 -- Install the Open-Module Driver from Backports:* |]]  nvidia-open-kernel-dkms  [[elisp:(org-shifttab)][<)]] E|
####+END

Install the *open* kernel module (required for Blackwell) plus the userspace driver:

#+begin_example
sudo apt install -t trixie-backports nvidia-open-kernel-dkms nvidia-driver
#+end_example

Notes:
- =nvidia-open-kernel-dkms= is the *open* (MIT/GPL) module --- do *not* use =nvidia-kernel-dkms=
  (the closed module) on this card.
- Installing the driver *blacklists =nouveau= automatically* (via =nvidia-kernel-common= /
  =glx-alternative-nvidia=). No manual blacklist file is normally needed.
- The GSP firmware Blackwell needs is pulled in as a dependency of the driver package.
- Confirm the DKMS module built for your kernel:

#+begin_example
dkms status | grep nvidia
apt-cache policy nvidia-driver   # confirm the installed version is 570.x or newer
#+end_example

/If =apt-cache policy= shows only =550.xx= (backports has no 570+ yet), stop here and use the
=.run= fallback section instead./

####+BEGIN: blee:bxPanel:foldingSection :outLevel 1 :title "Step 5 -- Secure Boot: Enroll Your MOK" :anchor "" :extraInfo "Sign the module"
* [[elisp:(show-all)][(>]]  _[[elisp:(blee:menu-sel:outline:popupMenu)][±]]_  _[[elisp:(blee:menu-sel:navigation:popupMenu)][Ξ]]_       [[elisp:(outline-show-subtree+toggle)][| *Step 5 -- Secure Boot: Enroll Your MOK:* |]]  Sign the module  [[elisp:(org-shifttab)][<)]] E|
####+END

With Secure Boot ON, an unsigned kernel module will *not load*. Debian's DKMS auto-generates a
*Machine Owner Key (MOK)* on first build and signs the module with it; you just have to *enroll the
public key* into the firmware once.

The key pair DKMS created:
- Private key :: =/var/lib/dkms/mok.key=
- Public cert :: =/var/lib/dkms/mok.pub=

/(If those files are missing, generate them with =sudo dkms generate_mok=.)/

Import the public key --- you will be asked to set a *one-time password* (8--16 chars) that you must
re-enter at the next boot:

#+begin_example
sudo mokutil --import /var/lib/dkms/mok.pub

# Confirm it is queued for enrollment
sudo mokutil --list-new
#+end_example

Now *reboot*. During boot the blue *MOK Manager* (Perform MOK management) screen appears:
- Choose *Enroll MOK* -> *Continue* -> *Yes*.
- Enter the *one-time password* you set above.
- *Reboot* again.

/The MOK Manager screen appears only once, right after the import. If you miss it, the machine boots
normally but the NVIDIA module stays unloaded --- just re-run the =mokutil --import= and reboot./

####+BEGIN: blee:bxPanel:foldingSection :outLevel 1 :title "Step 6 -- Reboot and Verify" :anchor "" :extraInfo "nvidia-smi"
* [[elisp:(show-all)][(>]]  _[[elisp:(blee:menu-sel:outline:popupMenu)][±]]_  _[[elisp:(blee:menu-sel:navigation:popupMenu)][Ξ]]_       [[elisp:(outline-show-subtree+toggle)][| *Step 6 -- Reboot and Verify:* |]]  nvidia-smi  [[elisp:(org-shifttab)][<)]] E|
####+END

After the MOK enrollment reboot, verify the driver is loaded and bound to the card:

#+begin_example
# Driver + card summary (this is the definitive "it works" check)
nvidia-smi

# The kernel module is loaded (and nouveau is NOT)
lsmod | egrep 'nvidia|nouveau'

# The card is now driven by nvidia, not nouveau
lspci -k -d 10de:2d04

# Confirm your MOK is enrolled
sudo mokutil --list-enrolled | grep -i dkms
#+end_example

=nvidia-smi= should print the GPU name (GeForce RTX 5060 Ti) and the driver version (570.x+).

####+BEGIN: blee:bxPanel:foldingSection :outLevel 1 :title "Fallback -- NVIDIA .run Installer (Open Modules)" :anchor "" :extraInfo "If backports lacks 570+"
* [[elisp:(show-all)][(>]]  _[[elisp:(blee:menu-sel:outline:popupMenu)][±]]_  _[[elisp:(blee:menu-sel:navigation:popupMenu)][Ξ]]_       [[elisp:(outline-show-subtree+toggle)][| *Fallback -- NVIDIA .run Installer (Open Modules):* |]]  If backports lacks 570+  [[elisp:(org-shifttab)][<)]] E|
####+END

Use this *only* if trixie-backports does not yet provide a 570+ driver. It works reliably on
Blackwell but lives *outside apt*, so it must be rebuilt/reinstalled after every kernel update.

- Download the current driver (570.x or newer) from https://www.nvidia.com/Download/index.aspx
  (or use the open-source module source at https://github.com/NVIDIA/open-gpu-kernel-modules).
- Purge any packaged NVIDIA driver first so the two do not collide:

#+begin_example
sudo apt purge '^nvidia-.*' && sudo apt autoremove
#+end_example

- Stop the graphical session and run the installer from a text console (=Ctrl-Alt-F3=), telling it
  to build the *open* modules and to *sign* them with your MOK for Secure Boot:

#+begin_example
sudo systemctl isolate multi-user.target
sudo sh ./NVIDIA-Linux-x86_64-570.*.run \
     --kernel-module-type=open \
     --module-signing-secret-key=/var/lib/dkms/mok.key \
     --module-signing-public-key=/var/lib/dkms/mok.pub
#+end_example

- =--kernel-module-type=open= is *mandatory* for Blackwell.
- If the MOK is not yet enrolled, enroll =/var/lib/dkms/mok.pub= as in Step 5, then reboot.
- Let the installer register with DKMS if it offers to, so it rebuilds on kernel upgrades.

/Per Debian's "DontBreakDebian" guidance, the packaged (backports) route is preferred; use =.run=
only as a stopgap until backports carries a Blackwell-capable driver./

####+BEGIN: blee:bxPanel:foldingSection :outLevel 1 :title "Troubleshooting" :anchor "" :extraInfo "Black screen, module not loading"
* [[elisp:(show-all)][(>]]  _[[elisp:(blee:menu-sel:outline:popupMenu)][±]]_  _[[elisp:(blee:menu-sel:navigation:popupMenu)][Ξ]]_       [[elisp:(outline-show-subtree+toggle)][| *Troubleshooting:* |]]  Black screen, module not loading  [[elisp:(org-shifttab)][<)]] E|
####+END

- *Black screen / boots to nouveau after install* :: The signed module is not loading. Almost always
  the MOK is not enrolled --- redo Step 5 and watch for the blue MOK Manager screen. Check with
  =sudo dmesg | egrep -i 'nvidia|nouveau|taint'=.
- *=nvidia-smi= says "No devices were found"* :: Usually a version too old for Blackwell. Confirm
  =570+= with =nvidia-smi= / =apt-cache policy nvidia-driver=; if it is =550=, you are still on the
  base-Trixie driver --- fix the backports pin (Step 2) or use the =.run= fallback.
- *DKMS module failed to build* :: Headers do not match the running kernel. Reboot into the backports
  kernel and reinstall matching =linux-headers-amd64= (Step 3); re-check =dkms status=.
- *Recover a broken graphical boot* :: At GRUB, boot with =nomodeset= (or pick an older kernel) to get
  a working console, then correct the driver/MOK and reboot.
- *Wayland vs Xorg* :: If the desktop misbehaves on Wayland with NVIDIA, try an Xorg session at the
  login screen (gear/settings icon) as a comparison point.
