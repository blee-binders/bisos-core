* [[elisp:(show-all)][(>]]  _[[elisp:(blee:menu-sel:outline:popupMenu)][±]]_  _[[elisp:(blee:menu-sel:navigation:popupMenu)][Ξ]]_       [[elisp:(outline-show-subtree+toggle)][| *Overview and Use Cases:* |]] <<Panel>>   /GeForce RTX 5060 Ti -- Debian 13/  [[elisp:(org-shifttab)][<)]] E|

This is the *parent* node for installing the *NVIDIA driver* for a *GeForce RTX 5060 Ti*
(Blackwell, =GB206=) on *Debian 13 (Trixie)*. It describes the situation and points to *two
child panels*, each documenting one installation route. Pick the child panel that applies to you.

*The key point up front:* the RTX 5060 Ti is a *Blackwell*-generation card. Blackwell GPUs require
an NVIDIA driver of the *570 series or newer* AND the *open kernel modules* (MIT/GPL). The driver
that ships in the base Debian 13 repository (=550.xx=) does *not* support Blackwell and will leave
you on =nouveau= or a black screen. So the ordinary =apt install nvidia-driver= recipe (as used in
the older =hwGraphicsNvidia= panel) *does not work on this card*. See the next section for details.

*Two routes, split into two child panels:*
- [[elisp:(blee:bnsm:panel-goto "/bisos/panels/bisos-core/bootstrap/hardwareBox/BlankBox/hwGraphics/nvidia-geforce-rtx-5060-ti/rtx-5060-ti-driver-nvidia/_nodeBase_")][ =rtx-5060-ti-driver-nvidia= ]]
  :: NVIDIA's official =.run= installer with =--kernel-module-type=open=, DKMS-built and MOK-signed.
  *This is the route that WORKS today* (verified on =intra=, 2026-07-05). Use this now.
- [[elisp:(blee:bnsm:panel-goto "/bisos/panels/bisos-core/bootstrap/hardwareBox/BlankBox/hwGraphics/nvidia-geforce-rtx-5060-ti/rtx-5060-ti-driver-debian/_nodeBase_")][ =rtx-5060-ti-driver-debian= ]]
  :: The Debian-native =apt=/=trixie-backports= open-module route. *Currently FAILED* (backports had
  only =550=); *kept for later* --- it is the preferred route once backports ships a 570+ driver.

*Common to both routes:* Secure Boot is left *enabled* (we enroll a Machine Owner Key / MOK so the
signed module loads), and the *NVIDIA card is treated as the primary GPU*. This machine also has
Intel *Arrow Lake-S* integrated graphics; see "Two Approaches" below for the dual-GPU choice.

/NOTE: This is a starting-point draft. Blackwell support on Debian is evolving fast; verify each
step and the current backports driver version against this specific machine before relying on it./

| Child panel                 | Route                                     | Status (2026-07-05, =intra=)        | Kernel-upgrade behaviour              |
|-----------------------------+-------------------------------------------+-------------------------------------+---------------------------------------|
| =rtx-5060-ti-driver-nvidia= | NVIDIA =.run= installer, open modules     | *WORKS* --- use this now            | manual/DKMS rebuild (outside apt)     |

*Recommendation:* use the =.run= child panel now; switch to the =apt=/backports child panel once
=apt-cache policy nvidia-driver= (with backports enabled) shows *570.x or newer*.

*About the two GPUs (NVIDIA + Intel Arrow Lake-S iGPU):* you do *not* need to disable the Intel iGPU.
The recommended options, in order:
- *Keep both enabled, NVIDIA as primary* (simplest with a desktop monitor on the NVIDIA card) --- the
  iGPU stays available and does no harm. This is what these panels assume.
- *PRIME render-offload* (hybrid): desktop runs on the iGPU, and you offload specific apps to the
  NVIDIA GPU with =prime-run= / =__NV_PRIME_RENDER_OFFLOAD=1=. Best for a power/heat trade-off ---
  documented step-by-step in the "Optional -- Dual-GPU PRIME Render-Offload" section of the
  =rtx-5060-ti-driver-nvidia= child panel.
- *Disable the iGPU in BIOS* :: only worth it if the iGPU actively causes boot/display confusion.
  Costs you the fallback console the iGPU provides if the NVIDIA driver ever breaks, so it is *not
  recommended* --- leaving it enabled is safer.

####+BEGIN: blee:bxPanel:foldingSection :outLevel 1 :title "This Card -- Why the Packaged Driver Fails" :anchor "" :extraInfo "Blackwell -- GB206"
* [[elisp:(show-all)][(>]]  _[[elisp:(blee:menu-sel:outline:popupMenu)][±]]_  _[[elisp:(blee:menu-sel:navigation:popupMenu)][Ξ]]_       [[elisp:(outline-show-subtree+toggle)][| *This Card -- Why the Packaged Driver Fails:* |]]  Blackwell -- GB206  [[elisp:(org-shifttab)][<)]] E|
####+END

The card, as reported by =lspci -nn= on the target machine:

#+begin_example
01:00.0 VGA compatible controller [0300]: NVIDIA Corporation GB206 [GeForce RTX 5060 Ti] [10de:2d04] (rev a1)
01:00.1 Audio device [0403]: NVIDIA Corporation GB206 High Definition Audio Controller [10de:22eb] (rev a1)
#+end_example

- *Architecture* :: Blackwell (=GB206=); PCI vendor:device =10de:2d04=.
- *Driver floor* :: Blackwell needs the *570-series* driver (or newer). Anything older simply has no
  device-ID match for =GB206= and will not drive the card.
- *Open modules are mandatory* :: For Turing and newer, NVIDIA offers two kernel-module flavors ---
  the legacy *proprietary* module and the *open* (MIT/GPL) module. For *Blackwell the open module is
  required*; the proprietary/closed kernel module refuses to initialize on these GPUs.
- *Why base Trixie fails* :: Debian 13's stock =nvidia-driver= is =550.xx= --- pre-Blackwell and
  closed-module. Hence we pull the open-module driver from *trixie-backports* (or use the =.run= fallback).

- Target machine (=intra=): Intel *Arrow Lake-S* iGPU (=8086:7d67=) + NVIDIA *GeForce RTX 5060 Ti* (=10de:2d04=).
- *2026-07-05 (=intra=) --- Primary (backports) path FAILED; must use the =.run= fallback:*
  - =trixie-backports= currently carries only =nvidia-driver 550.163.01=; there is *no 570+* yet.
    =550= has no device-ID for =GB206=, so it cannot drive this card (=nvidia-smi= reports
    "couldn't communicate with the NVIDIA driver").
  - =sudo apt full-upgrade= (with the =linux-*= pin) pulled in backports kernel =7.0.12+deb13=
    as the new default (running kernel at test time was still =6.12.94=). *Decision: stay on 7.0.12.*
  - DKMS built & signed the =550= open module for =6.12.94=, but *failed to compile for =7.0.12=*
    (=550= source is too old for the 7.0 kernel), leaving =nvidia-open-kernel-dkms= (=iF=) and
    =nvidia-driver= (=iU=) half-configured.
  - DKMS auto-generated the MOK key pair (=/var/lib/dkms/mok.key=, =mok.pub=); *MOK not yet enrolled*.
  - /Resolution: purge packaged nvidia, then install a 570+ open-module driver via the =.run=
    fallback (built against 7.0.12, signed with the MOK), enroll the MOK, reboot./
- /NOTYET: record the working 570+ =.run= version and =nvidia-smi= output once the fallback completes./
