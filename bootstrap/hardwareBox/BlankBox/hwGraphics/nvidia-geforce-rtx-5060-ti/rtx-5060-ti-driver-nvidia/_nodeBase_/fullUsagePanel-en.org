* [[elisp:(show-all)][(>]]  _[[elisp:(blee:menu-sel:outline:popupMenu)][±]]_  _[[elisp:(blee:menu-sel:navigation:popupMenu)][Ξ]]_       [[elisp:(outline-show-subtree+toggle)][| *Overview and Use Cases:* |]] <<Panel>>   /NVIDIA .run -- Open Modules -- WORKS/  [[elisp:(org-shifttab)][<)]] E|

This child panel documents the *working* way to drive a *GeForce RTX 5060 Ti*
(Blackwell, =GB206=) on *Debian 13 (Trixie)*: NVIDIA's official =.run= installer,
building the *open* kernel modules and signing them for Secure Boot.

*Why this route (and not apt):* the RTX 5060 Ti is *Blackwell*, which needs a *570-series
or newer* driver AND the *open* kernel modules. As of *2026-07-05* Debian =trixie-backports=
carried only =550.163.01= (no =GB206= support), so the apt route could not drive this card ---
see the sibling [[elisp:(blee:bnsm:panel-goto "/bisos/panels/bisos-core/bootstrap/hardwareBox/BlankBox/hwGraphics/nvidia-geforce-rtx-5060-ti/rtx-5060-ti-driver-debian/_nodeBase_")][ =rtx-5060-ti-driver-debian= ]] panel for that (kept-for-later) attempt.

*Route summary:*
- Confirm the card, then *purge* any packaged NVIDIA driver so the two do not collide.
- Download the current *570+* driver (this panel used =595.84=) into this =_nodeBase_=.
- Enroll a *MOK* (Secure Boot stays ON) so the signed module loads.
- Run the =.run= from a *text console* with =--kernel-module-type=open --dkms=, signing with the MOK.
- Reboot and verify with =nvidia-smi=.

*Trade-off:* the =.run= lives *outside apt*, so it must be rebuilt/reinstalled after every
kernel change. Registering it with DKMS (=--dkms=) makes it rebuild automatically on kernel upgrades.
Prefer the apt/backports route (sibling panel) once backports ships a Blackwell-capable driver.

/NOTE: On =intra= the running kernel is =7.0.12+deb13=; the modules are built against it. Re-verify
the driver version and =nvidia-smi= output on the specific machine before relying on this./

The card, as reported by =lspci -nn= on the target machine:

#+begin_example
01:00.0 VGA compatible controller [0300]: NVIDIA Corporation GB206 [GeForce RTX 5060 Ti] [10de:2d04] (rev a1)
01:00.1 Audio device [0403]: NVIDIA Corporation GB206 High Definition Audio Controller [10de:22eb] (rev a1)
#+end_example

- *Architecture* :: Blackwell (=GB206=); PCI vendor:device =10de:2d04=.
- *Driver floor* :: Blackwell needs the *570-series* driver (or newer). Older drivers have no
  device-ID match for =GB206= and will not drive the card.
- *Open modules mandatory* :: For Blackwell the *open* (MIT/GPL) kernel module is required; the
  closed/proprietary module refuses to initialize on these GPUs. Hence =--kernel-module-type=open=.

####+BEGIN: blee:bxPanel:foldingSection :outLevel 1 :title "Optional -- Dual-GPU PRIME Render-Offload" :anchor "" :extraInfo "iGPU drives desktop, offload apps to NVIDIA"
* [[elisp:(show-all)][(>]]  _[[elisp:(blee:menu-sel:outline:popupMenu)][±]]_  _[[elisp:(blee:menu-sel:navigation:popupMenu)][Ξ]]_       [[elisp:(outline-show-subtree+toggle)][| *Optional -- Dual-GPU PRIME Render-Offload:* |]]  iGPU drives desktop, offload apps to NVIDIA  [[elisp:(org-shifttab)][<)]] E|
####+END

*Optional --- only if you want the Intel iGPU to drive the desktop and use the NVIDIA GPU
per-application.* The default assumed by this panel is *NVIDIA-primary* (monitor on the NVIDIA card,
everything renders on NVIDIA); if that is your setup you can skip this section.

*What PRIME render-offload does:* the *Intel Arrow Lake-S iGPU* drives the monitors and the desktop
(compositor, browser, video), while the *NVIDIA GPU stays idle/powered-down* until you *explicitly*
launch a chosen app on it (a game, Blender, CUDA/AI work). That app renders on NVIDIA and the frames
are copied to the iGPU's framebuffer for display. This is the power/heat trade-off: the big GPU only
spins up on demand.

*Desktop caveat --- which GPU drives the monitor depends on the physical port:*
- Monitor in the *motherboard* video output -> the *iGPU* drives it -> PRIME offload applies (this section).
- Monitor in the *NVIDIA card's* DP/HDMI -> *NVIDIA* already drives the display -> you are effectively
  NVIDIA-primary and offload is unnecessary. Check with the provider list below.

*Prerequisites:* NVIDIA driver installed and loaded (Steps 1--6), =nvidia-drm.modeset=1= (default on
recent drivers), and the *iGPU set as the display/primary GPU*. Render-offload via these env vars is

*Install a =prime-run= wrapper.* Debian ships no =prime-run= of its own; =prime-run= is just a tiny
script that sets the NVIDIA offload environment variables before launching a program. Create it once

#!/bin/bash
# Create /usr/local/bin/prime-run -- a small wrapper for PRIME render-offload.
# It sets the NVIDIA offload env vars so a SINGLE app renders on the NVIDIA GPU
# while the Intel iGPU keeps driving the desktop. Debian has no prime-run of its
# own, so we install this one ourselves.
#
# Prereq: NVIDIA driver installed and loaded (this panel), iGPU as the display GPU.
# Usage after install:  prime-run <app>     e.g.  prime-run glxgears

sudo tee /usr/local/bin/prime-run > /dev/null <<'EOF'
#!/bin/sh
# Offload one app to the NVIDIA GPU (PRIME render-offload).
__NV_PRIME_RENDER_OFFLOAD=1 \
__GLX_VENDOR_LIBRARY_NAME=nvidia \
__VK_LAYER_NV_optimus=NVIDIA_only \
    "$@"
EOF

sudo chmod +x /usr/local/bin/prime-run
echo "Installed /usr/local/bin/prime-run"

- *Black screen / boots to nouveau after install* :: The signed module is not loading --- almost always
  the MOK is not enrolled. Redo Step 4 and watch for the blue MOK Manager screen. Check with
  =sudo dmesg | egrep -i 'nvidia|nouveau|taint'=.
- *After a kernel upgrade the module is gone* :: The =.run= is outside apt. If you used =--dkms= it
  rebuilds automatically; otherwise re-run =runInstallerFallback.sh= from a text console.
- *=nvidia-smi= says "No devices were found"* :: Driver too old for Blackwell --- confirm you installed
  a =570+= =.run=.
- *Recover a broken graphical boot* :: At GRUB boot with =nomodeset= (or an older kernel) to get a
  console, then correct the driver/MOK and reboot.
