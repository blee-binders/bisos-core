This child panel documents the *Debian-native* way to drive a *GeForce RTX 5060 Ti*
(Blackwell, =GB206=) on *Debian 13 (Trixie)*: the *open-module* driver
(=nvidia-open-kernel-dkms=) from *trixie-backports*, per Debian's "DontBreakDebian" guidance.

*Status: this route did NOT work on =intra= as of 2026-07-05 --- kept for later.* Backports carried
only =nvidia-driver 550.163.01=, which has *no device-ID for =GB206=* and cannot drive Blackwell;
DKMS also failed to build =550= against kernel =7.0.12=. Until backports ships a *570+* open-module
driver, use the working =.run= route in the sibling
[[elisp:(blee:bnsm:panel-goto "/bisos/panels/bisos-core/bootstrap/hardwareBox/BlankBox/hwGraphics/nvidia-geforce-rtx-5060-ti/rtx-5060-ti-driver-nvidia/_nodeBase_")][ =rtx-5060-ti-driver-nvidia= ]] panel.

*This route is preferred once backports is ready* --- it stays inside apt, so the driver rebuilds and
tracks kernel upgrades automatically (no manual reinstall after each kernel change).

*Route summary:* enable =non-free-firmware= + =trixie-backports= and pin -> install matching kernel
headers -> install =nvidia-open-kernel-dkms nvidia-driver= from backports -> enroll a MOK (Secure Boot
ON) -> reboot and verify.

####+BEGIN: blee:bxPanel:foldingSection :outLevel 1 :title "This Card -- Why the Packaged Driver Fails" :anchor "" :extraInfo "Blackwell -- GB206"
* [[elisp:(show-all)][(>]]  _[[elisp:(blee:menu-sel:outline:popupMenu)][±]]_  _[[elisp:(blee:menu-sel:navigation:popupMenu)][Ξ]]_       [[elisp:(outline-show-subtree+toggle)][| *This Card -- Why the Packaged Driver Fails:* |]]  Blackwell -- GB206  [[elisp:(org-shifttab)][<)]] E|
####+END

The card, as reported by =lspci -nn= on the target machine:

#+begin_example
01:00.0 VGA compatible controller [0300]: NVIDIA Corporation GB206 [GeForce RTX 5060 Ti] [10de:2d04] (rev a1)
01:00.1 Audio device [0403]: NVIDIA Corporation GB206 High Definition Audio Controller [10de:22eb] (rev a1)
#+end_example

- *Architecture* :: Blackwell (=GB206=); PCI vendor:device =10de:2d04=.
- *Driver floor* :: Blackwell needs the *570-series* driver (or newer); older drivers have no device-ID
  match for =GB206=.
- *Open modules mandatory* :: For Blackwell the *open* (MIT/GPL) module is required --- use
  =nvidia-open-kernel-dkms=, *not* =nvidia-kernel-dkms= (closed).
- *Why base Trixie fails* :: Debian 13's stock =nvidia-driver= is =550.xx= --- pre-Blackwell and
  closed-module. So we pull the open-module driver from *trixie-backports* (when it carries 570+).

####+BEGIN: blee:bxPanel:foldingSection :outLevel 1 :title "Prerequisites and Assumptions" :anchor "" :extraInfo "Debian 13 -- UEFI -- Secure Boot ON"
* [[elisp:(show-all)][(>]]  _[[elisp:(blee:menu-sel:outline:popupMenu)][±]]_  _[[elisp:(blee:menu-sel:navigation:popupMenu)][Ξ]]_       [[elisp:(outline-show-subtree+toggle)][| *Prerequisites and Assumptions:* |]]  Debian 13 -- UEFI -- Secure Boot ON  [[elisp:(org-shifttab)][<)]] E|
####+END

- A working *Debian 13 (Trixie)* install, booted in *UEFI* mode, with network access.
- *Secure Boot ON* --- we enroll a MOK so the signed module loads (Step 5).
- =sudo=/root access; a few hundred MB free for the DKMS builds.
- A plan for console access (=Ctrl-Alt-F3=, or another machine + SSH) if X/Wayland fails to start.
