* Overview

=docker-systemctl-replacement= is a Python script that replaces =/usr/bin/systemctl= so you can run systemd services inside Docker containers without requiring a full systemd daemon.

This is useful for:
- Testing deployment scripts in containers
- Running multiple services in a single container (like a light-weight VM)
- Allowing Ansible and other deployment tools to work with containers seamlessly
- Automatic service startup/shutdown and zombie reaping

** Key Features

- Service manager: start/stop/enable/disable services
- Init daemon: automatically start enabled services at container boot
- Zombie reaper: reaps orphaned processes
- Drop-in replacement for systemctl
- Parses *.service files from standard packages

* Installation

** Addition Inside of the Container --- Does not Become PID 1

#+BEGIN_SRC bash
pushd /bisos/git/anon/ext/misc/docker-systemctl-replacement ; git clone https://github.com/gdraheim/docker-systemctl-replacement.git ; popd
sudo mv /usr/bin/systemctl /usr/bin/systemctl.dist
sudo cp  /bisos/git/anon/ext/misc/docker-systemctl-replacement/files/docker/systemctl3.py /usr/bin/systemctl
sudo chmod 755 /usr/bin/systemctl
sudo mv /usr/bin/journalctl /usr/bin/journalctl.dist
sudo cp  /bisos/git/anon/ext/misc/docker-systemctl-replacement/files/docker/journalctl3.py /usr/bin/journalctl
sudo chmod 755 /usr/bin/journalctl
mkdir -p /run/systemd/system
#+END_SRC


** In Dockerfile

#+BEGIN_SRC dockerfile
FROM debian:bookworm
RUN apt-get update && apt-get install -y python3 curl
RUN curl -o /usr/bin/systemctl \
  https://raw.githubusercontent.com/gdraheim/docker-systemctl-replacement/master/files/docker/systemctl3.py
RUN chmod +x /usr/bin/systemctl
RUN mkdir -p /run/systemd/system
#+END_SRC

#+BEGIN_SRC
docker stop --time 100 running
docker run --stop-timeout 100 --name running image
docker-compose.yml: stop_grace_period: 100
#+END_SRC

* Service Files

Services are defined in standard systemd *.service files, typically located in:
- =/etc/systemd/system/=
- =/usr/lib/systemd/system/=

The script parses ExecStart and ExecStop directives to manage services.

* Container Entrypoint Mode

To use as the main container init (PID 1) with automatic service startup:

** Dockerfile Configuration

#+BEGIN_SRC dockerfile
FROM debian:bookworm

# Install dependencies
RUN apt-get update && apt-get install -y python3 curl

# Install systemctl replacement
RUN curl -o /usr/bin/systemctl \
  https://raw.githubusercontent.com/gdraheim/docker-systemctl-replacement/master/files/docker/systemctl3.py
RUN chmod +x /usr/bin/systemctl

# Create systemd directory structure
RUN mkdir -p /run/systemd/system

# Copy or install your service definitions
# (e.g., COPY myservice.service /etc/systemd/system/)

# Set as entrypoint
ENTRYPOINT ["/usr/bin/systemctl"]

# Default: start multi-user.target (all enabled services)
CMD ["default"]
#+END_SRC

** Running the Container

#+BEGIN_SRC bash
# Container starts, systemctl becomes PID 1 and auto-starts all enabled services
docker run myimage

# From another terminal, check running services
docker exec mycontainer systemctl list-units --state=running

# Graceful shutdown: all services are stopped cleanly
docker stop mycontainer
#+END_SRC

** Behavior

When run as PID 1 entrypoint:
- Automatically starts all services marked with =systemctl enable=
- Automatically stops all services when container receives SIGTERM (docker stop)
- Reaps zombie processes (handles PID 1 reaping responsibility)
- Provides systemctl interface for deployment automation

* Documentation and Resources

- **GitHub**: https://github.com/gdraheim/docker-systemctl-replacement
- **PyPI**: https://pypi.org/project/docker-systemctl-replacement/
- **LICENSE**: EUPL-1.2 (permissive, allows copying to projects)

* Common Issues and Solutions

| Issue                      | Solution                                          |
|----------------------------+---------------------------------------------------|
| Service doesn't start      | Check *.service file ExecStart directive syntax   |
| Zombie processes persist   | Ensure systemctl is running as PID 1 (ENTRYPOINT) |
| Services not auto-starting | Run systemctl enable for each service             |
| Ansible connection fails   | Verify ansible_connection: docker in inventory    |
